# Rules

AI-readable project rules for this repository. All AI assistants should read these before working on the codebase.

## Organization

Rules are numbered by category:

- `01-tech-stack.md` — Stack, versions, key libraries
- `02-architecture.md` — System design, data flow, deployment
- `03-file-conventions.md` — Where files go, naming patterns
- `04-coding-standards.md` — Code style, patterns, libraries
- `05-workflow.md` — How to approach tasks

## Adding Rules

Create a new numbered file (e.g., `06-testing.md`) or add to an existing file if it fits. Keep rules concise and actionable.
